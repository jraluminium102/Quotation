# SPEC — เครื่องคิดราคา R3.9 (Price Calculator) เข้าระบบ Quotation

> **สถานะ:** v1 · **อัปเดต:** 2026-06-02 · **เจ้าของ:** พี่นัท (CEO)
> **Source:** `00_ล่าสุด_ส่งUX/JR_คิดราคา_R3.9_UX.html` (เครื่องคิดราคา) + `R3.9_v11_ล่าสุด_ตรวจ+ไฮไลท์.docx` (สูตรอ้างอิง ตรวจเทียบ R1 แล้ว)
> **กฎเหล็ก:** สูตร/rate table = ของจริงที่ตรวจแล้ว — **ห้าม re-derive/เดา** ⇒ reuse ตัวคำนวณเดิม

## 1. Problem Statement

เซลล์คิดราคางานอลู-กระจกด้วยเครื่องคิดราคา R3.9 (ไฟล์ HTML แยกเดี่ยว) แล้วค่อยมาพิมพ์ใบเสนอราคาในระบบซ้ำอีกที — กรอกซ้ำ เสี่ยงเลขไม่ตรงกับที่คำนวณ และเครื่องคิดราคาอยู่นอกระบบ ERP (เปิดไฟล์เอง ไม่มี auth ไม่ผูกกับงาน) ทำให้ราคาที่เสนอกับที่คำนวณหลุดจากกันได้

## 2. Goals

| # | เป้าหมาย | วัดผล |
|---|---------|-------|
| G1 | เครื่องคิดราคาอยู่ในระบบ ERP เดียว (หลัง login) | เปิด `/calculator` ในแอปได้ ไม่ต้องเปิดไฟล์แยก |
| G2 | รักษาสูตร R3.9 ที่ตรวจแล้ว 100% | ผลคำนวณตรงกับไฟล์เดิมทุกเคส (ไม่แตะสูตร) |
| G3 | (P1 ถัดไป) ส่งผลคิดราคา → สร้างใบเสนอราคาไม่กรอกซ้ำ | line item เข้า `/quotations/new` อัตโนมัติ |

## 3. Non-Goals (v1 นี้)

- **ไม่ rewrite สูตรราคาใหม่ใน TS** — เสี่ยงพิมพ์ rate table ผิด (มีหลายสิบช่วง) ใช้ตัวเดิมที่ verify แล้ว
- ไม่แก้ตรรกะ/หน้าตาเครื่องคิดราคา R3.9 (ยกมาทั้งดุ้น)
- ไม่ทำ data-bridge เข้าใบเสนอใน v1 นี้ (เป็น P1 สเต็ปถัดไป — ดูข้อ 6.2)
- ไม่ย้าย owner cost/margin ไปคุมฝั่ง server (P2 — ดู Open Questions)

## 4. ขอบเขตสูตร (อ้างอิง R3.9 v11 — ห้ามเดา)

สรุปจากเอกสารอ้างอิง (ตัวเลขจริงอยู่ในเครื่องคิดราคา):
- **1.1 บานเลื่อนภายนอก SMS** — `max(6,500, พื้นที่×rate)` (rate ตามช่วง ตร.ม.)
- **1.2 บานเลื่อนภายนอก ยูโร** — `max(7,500, พื้นที่×rate)`
- **2 บานกระจกติดตาย** — `max(5,000, พื้นที่×rate)`
- **3 บานกระทุ้ง ยูโร** — `max(10,800, max(0.6,พื้นที่)×rate) + บานเพิ่ม×2,900` + option (เปิดล่าง/ข้าง/tilt&turn, แขนค้ำ, มอเตอร์)
- **4 ประตูบานเปิด ยูโร** — เดี่ยว `max(18,000,…)` / คู่ `max(32,000,…)` + option
- **5 บานเฟี้ยม, มุ้ง (หลายชนิด), กระจก, ระแนง/รั้ว/ราวกันตก** — ตาม R3.9
- **ค่าบริการเพิ่มเติม (ทั้งใบ):** ติดตั้ง/ขนส่ง/protection/เดินทาง
- **Owner mode:** ต้นทุน/กำไร (Margin/Profit) — โชว์เฉพาะเจ้าของ

## 5. User Stories

- **เซลล์:** เปิดเครื่องคิดราคาในระบบหลัง login เพื่อคิดราคาได้ทันที ไม่ต้องหาไฟล์
- **เซลล์:** (P1) กดส่งผลคิดราคาเข้าใบเสนอราคา เพื่อไม่กรอกรายการซ้ำ
- **เจ้าของ:** ดูต้นทุน/กำไรในเครื่องคิดราคา (โหมดเจ้าของ) โดยลูกค้าไม่เห็น
- **Edge:** เปิดบนมือถือได้ · export Excel ได้ (ต้องมีเน็ตสำหรับ SheetJS CDN)

## 6. Requirements

### 6.1 Must-Have (P0) — v1 นี้
- [ ] เครื่องคิดราคา R3.9 เข้าถึงได้ที่เมนู **“เครื่องคิดราคา”** หลัง login
- [ ] ผลคำนวณ/หน้าตา/option เหมือนไฟล์เดิมทุกอย่าง (ยกมาทั้งไฟล์)
- [ ] อยู่ภายใต้ shell เดียวกับระบบ (sidebar + auth ป้องกันหน้า iframe)
- [ ] Build + deploy บน Vercel ได้

**Acceptance:**
- Given login แล้ว · When กดเมนูเครื่องคิดราคา · Then เห็นเครื่องคิดราคา R3.9 เต็มรูปแบบในแอป
- Given กรอกบานเลื่อน SMS พื้นที่ 3.0 ตร.ม. · Then ราคาตรงกับไฟล์เดิม (สูตรเดียวกัน)

### 6.2 Nice-to-Have (P1) — สเต็ปถัดไป
- ปุ่ม **“ส่งไปสร้างใบเสนอราคา”** ในเครื่องคิดราคา → ส่ง line items (รายการ, จำนวน, ราคา/หน่วย) เข้า `/quotations/new` ผ่าน `postMessage` → เติมฟอร์มอัตโนมัติ (เติม PRD P0-8 “เชื่อม R3.9”)
- ดึงข้อมูลลูกค้าจากเครื่องคิดราคา → map กับ Customer Master

### 6.3 Future (P2)
- พอร์ตเป็น React component เต็มตัว (เลิก iframe) + กด owner cost/margin จาก role จริง (server-gated)
- เก็บผลคิดราคาแต่ละครั้งลง DB ผูกกับงาน

## 7. Build Approach (ตัดสินแล้ว)

**Integrate (ไม่ rewrite):**
1. คัดลอกไฟล์เครื่องคิดราคา → `public/calculator/index.html` (เสิร์ฟ static)
2. หน้า `(app)/calculator/page.tsx` — iframe ความสูงเต็มจอ อยู่ใน shell (auth ป้องกันหน้า)
3. เพิ่มเมนู “เครื่องคิดราคา” ใน Sidebar

**เหตุผล:** สูตร R3.9 ผ่านการตรวจเทียบ R1 แล้ว (docx) — การพิมพ์ rate table ใหม่ใน TS เสี่ยงผิด ผิดกฎเหล็ก “ห้ามเดาราคา” การ reuse คือทางที่ถูกต้องและเร็วสุด
